# FillDoc

FillDoc - настольное Windows-приложение для подготовки юридических документов по Word-шаблонам. Проекты и их поля хранятся в Excel, а FillDoc подставляет значения из выбранной строки проекта в переменные внутри `.docx`-шаблонов.

## Что умеет проект

- читает карточки проектов из Excel-файла;
- сканирует папку Word-шаблонов и находит переменные вида `{Название поля}`;
- показывает, какие данные уже есть в проекте, а какие нужно дозаполнить;
- генерирует готовые `.docx` в выбранную папку выгрузки;
- хранит пользовательские настройки локально на компьютере;
- собирается в `.exe` через PyInstaller.

## Требования

- Windows 10/11;
- Python 3.11 или новее;
- доступный из PowerShell `python`;
- Microsoft Word не обязателен для генерации, но нужен для редактирования шаблонов.

Проверка Python:

```powershell
python --version
```

Если PowerShell пишет, что `python` не найден, установите Python с [python.org](https://www.python.org/downloads/windows/) и включите опцию `Add python.exe to PATH`, затем откройте PowerShell заново.

## Установка окружения

Из корня репозитория:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
python -m pip install -e ".[dev]"
python -m pytest
```

Команда `pip install -e ".[dev]"` ставит FillDoc в editable-режиме, основные зависимости приложения и инструменты разработки.

## Запуск

После активации окружения приложение можно запустить любым из двух способов:

```powershell
filldoc
```

или:

```powershell
python main.py
```

В приложении откройте вкладку настроек и укажите:

- Excel-файл проектов;
- папку шаблонов Word;
- папку выгрузки готовых документов;
- папку документов проектов, если она используется в вашем процессе.

## Excel-файл проектов

FillDoc ожидает обычный `.xlsx` или `.xlsm` файл:

- первая строка листа - заголовки колонок;
- каждая следующая непустая строка - отдельный проект;
- основной лист называется `Текущие`; если такого листа нет, используется первый лист книги;
- лист `Архив` используется для архивных проектов, если он есть;
- идентификатор проекта берется из колонки с номером дела, например `№ дела`, `№дела`, `Номер дела`, `Номер осн. дела`; если такой колонки нет или значение пустое, используется служебный идентификатор `row:<номер строки>`.

Названия колонок важны: именно они сопоставляются с переменными Word-шаблонов. Например, колонка `Должник` может заполнять переменную `{Должник}`.

Перед записью изменений в Excel приложение создает резервные копии рядом с Excel-файлом в папке `_filldoc_backups`.

## Word-шаблоны

Шаблоны должны быть файлами `.docx`. FillDoc рекурсивно сканирует выбранную папку шаблонов и ее подпапки.

Переменные пишутся в фигурных скобках:

```text
{Должник}
{Заказчик}
{ИНН должника}
```

Правила:

- внутри скобок пишется имя поля без дополнительных фигурных скобок;
- имя должно совпадать с колонкой Excel или с вариантом из словаря переменных;
- переменные поддерживаются в обычных абзацах, таблицах, колонтитулах и вложенных таблицах;
- временные файлы Word с именами `~$...` игнорируются;
- старый формат `.doc` не поддерживается.

При сканировании шаблонов FillDoc создает служебные карточки в подпапке `.filldoc` внутри папки шаблонов. Там сохраняется кэш найденных переменных и правило имени выходного файла.

## Сборка `.exe`

Сборка выполняется через PyInstaller. После установки dev-зависимостей можно запустить:

```powershell
.\build.ps1
```

Результат появится в:

```text
dist\FillDoc\FillDoc.exe
```

Если PyInstaller не установлен, `build.ps1` попробует установить его автоматически. При ручной сборке можно использовать файл `FillDoc.spec`.

## Настройки

Пользовательские настройки хранятся вне репозитория:

```text
%USERPROFILE%\.filldoc\settings.json
```

В файле сохраняются пути к Excel, шаблонам, папке выгрузки, папке документов, индивидуальные папки документов проектов и выбранная тема интерфейса.

# Архитектура FillDoc

FillDoc - локальное desktop-приложение на Python и PySide6. Основной сценарий: пользователь выбирает проект из Excel, выбирает один или несколько Word-шаблонов, приложение сопоставляет переменные шаблонов с полями проекта и сохраняет заполненные `.docx` в папку выгрузки.

## Точка входа

- `main.py` - совместимая точка запуска из корня репозитория.
- `src/filldoc/app.py` - основная точка входа пакета.
- `pyproject.toml` объявляет console script `filldoc = filldoc.app:run`.

## Основные слои

### UI

Папка: `src/filldoc/ui/`

Отвечает за окна, вкладки, тему и пользовательские действия. Главное окно собирает вкладки:

- настройки путей и темы;
- список проектов;
- библиотека шаблонов;
- словарь/переменные;
- заполнение документов.

UI не должен напрямую знать детали формата `.docx` или `.xlsx` глубже, чем нужно для отображения результата. Для работы с данными он вызывает сервисные классы из `excel`, `templates`, `fill` и `generator`.

### Настройки

Папка: `src/filldoc/core/`

`AppSettings` загружает и сохраняет пользовательские настройки в:

```text
%USERPROFILE%\.filldoc\settings.json
```

В настройках хранятся:

- путь к Excel-файлу проектов;
- путь к папке шаблонов;
- путь к папке выгрузки;
- путь к папке документов;
- индивидуальные папки документов проектов;
- тема интерфейса.

### Excel

Папка: `src/filldoc/excel/`

`ExcelProjectStore` работает с Excel через `openpyxl`:

- читает проекты из листа `Текущие` или первого листа книги;
- читает и поддерживает лист `Архив`;
- сохраняет изменения обратно в Excel;
- создает резервные копии перед записью;
- пропускает пустые строки и случайно продублированные строки заголовков.

Модель проекта находится в `src/filldoc/excel/models.py`. Проект - это строка Excel, представленная как `fields: dict[str, str]`, плюс служебные поля `project_id`, `headers` и `row_index`.

### Шаблоны

Папка: `src/filldoc/templates/`

`TemplateLibrary` рекурсивно сканирует папку шаблонов, находит `.docx`, извлекает переменные и создает карточки шаблонов. Служебные карточки сохраняются в подпапку `.filldoc` внутри папки шаблонов.

Переменные извлекаются из текста абзацев и таблиц по формату:

```text
{Имя переменной}
```

### Переменные

Папка: `src/filldoc/variables/`

Словарь переменных нормализует разные варианты написания одного поля. Например, шаблон может использовать одно имя, а Excel - близкое по смыслу другое имя. Это позволяет постепенно приводить шаблоны и таблицы к единому стандарту.

### Генерация документов

Папка: `src/filldoc/generator/`

Генератор открывает `.docx` через `python-docx`, заменяет плейсхолдеры на значения и сохраняет результат в папку выгрузки. Поддерживаются:

- обычные абзацы;
- таблицы;
- вложенные таблицы;
- колонтитулы.

Имена файлов очищаются от недопустимых для Windows символов. Если файл уже существует, новый файл получает суффикс вида `(2)`.

## Тесты

Тесты лежат в `tests/`. Базовая команда:

```powershell
python -m pytest
```

Сейчас тестами покрыты отдельные правила: чтение Excel, ротация бэкапов, извлечение переменных, словарь переменных и правила именования выходных файлов.

## Сборка

Сборка `.exe` описана в:

- `build.ps1`;
- `FillDoc.spec`.

Основной артефакт после сборки:

```text
dist\FillDoc\FillDoc.exe
```

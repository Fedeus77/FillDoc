# Roadmap

Этот файл фиксирует направление развития FillDoc. Он не заменяет задачи в трекере, но помогает понимать, в каком порядке улучшать проект.

## Этап 1. Зафиксировать базу

Цель: проект можно открыть, установить, запустить, проверить и понять.

Состав:

- нормальная установка через `python -m pip install -e .[dev]`;
- запуск тестов через `python -m pytest`;
- README в UTF-8;
- базовая документация в `docs/`;
- описание Excel, Word-шаблонов, настроек и сборки.

## Этап 2. Привести кодировку и тексты к порядку

В коде остаются строки и комментарии с поврежденной кириллицей. Нужно:

- восстановить пользовательские тексты интерфейса;
- восстановить комментарии там, где они полезны;
- проверить словарь переменных;
- добавить тесты на ключевые русскоязычные имена полей.

## Этап 3. Укрепить модель данных

Цель: меньше зависеть от случайных названий колонок и ручных совпадений.

Возможные задачи:

- единый реестр стандартных полей;
- явная схема Excel;
- импорт/валидация Excel-файла;
- понятные сообщения о несовпадениях;
- миграция старых названий колонок.

## Этап 4. Улучшить работу с шаблонами

Возможные задачи:

- редактор карточки шаблона;
- предпросмотр найденных переменных;
- отчет о незаполненных переменных;
- проверка шаблонов на ошибки;
- стабильные правила именования выходных файлов.

## Этап 5. Надежная генерация

Возможные задачи:

- больше тестов для замены переменных в сложных `.docx`;
- поддержка разрывов, секций и нестандартных элементов Word;
- отчет о каждом созданном файле;
- отдельный журнал ошибок генерации.

## Этап 6. Упаковка и релизы

Возможные задачи:

- автоматическая сборка `.exe`;
- версия приложения в интерфейсе;
- release notes;
- smoke-тест собранного приложения;
- инструкция для установки на рабочие компьютеры.

## Этап 7. Пользовательский комфорт

Возможные задачи:

- улучшенная навигация по проектам;
- поиск и фильтры;
- понятные статусы заполненности;
- быстрый переход к папкам проекта;
- сохранение пользовательских предпочтений.

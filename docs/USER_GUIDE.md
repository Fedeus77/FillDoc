# Руководство пользователя FillDoc

## Назначение программы

FillDoc заполняет Word-шаблоны данными из Excel-файла проектов.

Программа:

- читает список проектов из Excel;
- сканирует папку с `.docx`-шаблонами;
- находит переменные вида `{Название поля}`;
- показывает, каких данных не хватает;
- формирует готовые `.docx`-документы в папке выгрузки.

## Первый запуск

1. Установите зависимости по инструкции из `README.md`.
2. Запустите приложение:

```powershell
filldoc
```

или:

```powershell
python main.py
```

3. Откройте вкладку настроек.
4. Укажите:
   - Excel-файл проектов;
   - папку Word-шаблонов;
   - папку выгрузки;
   - папку документов проектов, если она используется в вашем процессе.
5. Сохраните настройки.
6. Вернитесь в основные вкладки и обновите данные.

## Подготовка Excel

FillDoc ожидает обычный `.xlsx` или `.xlsm` файл.

Рекомендуемая структура:

- лист `Текущие` - активные проекты;
- лист `Архив` - архивные проекты;
- первая строка листа - заголовки колонок;
- каждая следующая непустая строка - отдельный проект.

Минимальный пример колонок:

```text
№ дела | Заказчик | Должник | ИНН должника | Дата решения
```

Названия колонок используются как имена полей для шаблонов. Если в шаблоне указано `{Должник}`, FillDoc будет искать значение в колонке `Должник` или в словаре переменных.

## Подготовка шаблонов

Шаблоны должны быть файлами `.docx`. Их можно раскладывать по подпапкам.

Пример структуры:

```text
templates\
  Договоры\
    Уступка.docx
  Заявления\
    Заявление в суд.docx
```

Внутри шаблона переменные пишутся в фигурных скобках:

```text
Должник: {Должник}
ИНН: {ИНН должника}
Заказчик: {Заказчик}
```

Правила:

- внутри скобок должно быть имя поля без лишних символов;
- имя должно совпадать с колонкой Excel или быть описано в словаре;
- временные файлы Word `~$...` игнорируются;
- формат `.doc` не поддерживается.

## Работа со словарём переменных

Словарь переменных нужен для того, чтобы FillDoc понимал, что разные написания могут означать одно и то же поле.

Пример:

```text
{ИНН должника}
{ИНН ДОЛЖНИКА}
{ИНН Должник}
```

Для программы без словаря это три разных переменных. Со словарём это одна переменная, связанная с одним полем Excel.

Пользовательский словарь хранится в файле:

```text
%USERPROFILE%\.filldoc\variables.json
```

Обычно вручную этот файл редактировать не нужно. Используйте раздел:

```text
Переменные -> Словарь
```

### Что есть в разделе

В левой части отображается список переменных словаря. Для каждой записи видны:

- название;
- техническое имя;
- тип поля;
- обязательность.

В правой части находится карточка выбранной переменной:

- техническое имя;
- отображаемое название;
- группа;
- тип поля;
- обязательность;
- список aliases;
- комментарий;
- использование в шаблонах.

### Как добавить новую переменную

1. Откройте вкладку `Переменные`.
2. Перейдите во вкладку `Словарь`.
3. Нажмите кнопку `+`.
4. Заполните карточку.
5. Нажмите кнопку сохранения.

Пример:

```text
Техническое имя: inn_debtor
Название: ИНН должника
Группа: debtor
Тип поля: text
Обязательная: да
Комментарий: ИНН юрлица или ИП
```

Рекомендация:

- техническое имя делайте коротким и стабильным;
- отображаемое название делайте понятным для пользователя;
- группу используйте для логической сортировки, например `debtor`, `customer`, `case`.

### Как добавить alias

Alias - это дополнительный вариант написания той же переменной.

Пример: если в разных шаблонах встречаются `ИНН ДОЛЖНИКА` и `ИНН Должник`, их нужно добавить как aliases к переменной `ИНН должника`.

Порядок действий:

1. Выберите переменную в списке.
2. В поле `Новый alias` введите новый вариант.
3. Нажмите кнопку `+`.
4. Убедитесь, что alias появился в списке.
5. Сохраните словарь.

Пример aliases:

```text
ИНН ДОЛЖНИКА
ИНН Должник
инн должника
```

После сохранения FillDoc будет считать эти написания одной переменной.

### Когда создавать новую переменную

Создавайте новую переменную, если это действительно новое поле по смыслу.

Примеры:

```text
Дата договора цессии
Номер исполнительного листа
Адрес регистрации должника
```

### Когда добавлять alias

Добавляйте alias, если поле уже существует, но в шаблоне написано иначе.

Примеры:

```text
Должник
ДОЛЖНИК
Наименование должника
```

Если это одно и то же значение, нужна одна переменная и несколько aliases.

### Тип поля

Тип поля описывает, какие данные ожидаются:

- `text` - обычный текст;
- `multiline` - длинный текст, адрес, резолютивная часть;
- `select` - значение из набора вариантов;
- `date` - дата;
- `number` - число или сумма;
- `boolean` - да/нет.

Сейчас тип сохраняется в словаре как описание переменной. В дальнейшем он может использоваться для более строгих проверок и более удобного ввода.

### Обязательные переменные

Флаг `Обязательная` показывает, что поле важно для документа.

Используйте его для ключевых данных:

- номер дела;
- должник;
- ИНН;
- дата решения;
- сумма;
- номер договора;
- другие критичные реквизиты.

### Использование в шаблонах

В блоке `Использование в шаблонах` показывается, где используется выбранная переменная.

Это нужно, чтобы понимать:

- какие шаблоны зависят от переменной;
- можно ли менять название;
- где используется старый alias;
- почему переменная попадает в анализ недостающих полей.

Чтобы этот список был актуальным, в настройках должна быть указана папка шаблонов.

### Как словарь влияет на генерацию

Если в Excel есть колонка:

```text
ИНН должника
```

а в шаблоне написано:

```text
{ИНН ДОЛЖНИКА}
```

то после добавления `ИНН ДОЛЖНИКА` в aliases FillDoc возьмёт значение из поля `ИНН должника`.

Словарь используется при:

- анализе качества шаблонов;
- поиске недостающих полей;
- заполнении одного шаблона;
- пакетной генерации документов.

## Работа с шаблонами

Во вкладке `Шаблоны` программа:

- сканирует папку шаблонов;
- показывает найденные документы;
- извлекает переменные;
- показывает заполненные и недостающие поля;
- позволяет сформировать один документ или пакет документов.

Для работы:

1. Откройте вкладку `Шаблоны`.
2. Нажмите обновление данных.
3. Выберите проект.
4. Выберите шаблон или несколько шаблонов.
5. Проверьте список переменных.
6. При необходимости заполните недостающие поля.
7. Запустите генерацию.

## Генерация документов

Обычный сценарий:

1. Выберите проект.
2. Выберите шаблон.
3. Проверьте, какие поля уже заполнены.
4. Заполните недостающие поля.
5. Нажмите `Заполнить шаблон`.

Для пакетной генерации:

1. Отметьте несколько активных шаблонов.
2. Нажмите `Анализ пакета`.
3. Заполните общие недостающие поля.
4. Нажмите `Сгенерировать пакет`.

Готовые документы появятся в папке выгрузки из настроек.

## Работа с настройками

Настройки сохраняются локально:

```text
%USERPROFILE%\.filldoc\settings.json
```

Если приложение не видит Excel, шаблоны или папку выгрузки:

1. откройте настройки;
2. проверьте пути;
3. сохраните настройки повторно;
4. обновите данные во вкладках.

## Резервные копии Excel

Перед записью изменений FillDoc создаёт резервные копии Excel-файла в папке:

```text
_filldoc_backups
```

Папка создаётся рядом с исходным Excel-файлом.

Это защита от потери данных при:

- сохранении изменений;
- архивировании проектов;
- удалении проектов;
- конфликте записи.

## Частые проблемы

### PowerShell не находит Python

Проверьте:

```powershell
python --version
```

Если команда не работает, установите Python 3.11+ и добавьте его в `PATH`.

### Excel не сохраняется

Причины обычно две:

- файл открыт в Microsoft Excel;
- у пользователя нет прав записи.

Закройте Excel-файл и повторите действие.

### Шаблон не отображается

Проверьте:

- расширение файла должно быть `.docx`;
- имя файла не должно начинаться с `~$`;
- файл должен лежать внутри выбранной папки шаблонов.

### Переменная не заполнилась

Проверьте:

- имя внутри `{...}` совпадает с колонкой Excel;
- либо это имя есть в словаре;
- либо оно добавлено как alias к существующей переменной.

### Не отображается использование переменной в шаблонах

Проверьте, что:

- в настройках указана корректная папка шаблонов;
- шаблоны были пересканированы;
- переменная или её alias действительно встречается в `.docx`.

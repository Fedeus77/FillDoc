# Руководство пользователя

## Первый запуск

1. Установите зависимости по инструкции из `README.md`.
2. Запустите приложение:

```powershell
filldoc
```

или:

```powershell
python main.py
```

3. Откройте вкладку настроек.
4. Укажите Excel-файл проектов.
5. Укажите папку Word-шаблонов.
6. Укажите папку выгрузки готовых документов.
7. Сохраните настройки и проверьте пути.

## Подготовка Excel

Excel-файл должен содержать строку заголовков и строки проектов. Рекомендуемая структура:

- лист `Текущие` - активные проекты;
- лист `Архив` - архивные проекты;
- первая строка каждого листа - заголовки;
- одна строка ниже заголовков - один проект.

Минимальный пример колонок:

```text
№ дела | Заказчик | Должник | ИНН должника | Дата решения
```

Названия колонок используются как имена полей для шаблонов.

## Подготовка шаблонов

Шаблоны должны быть файлами `.docx`. Их можно разложить по подпапкам, например:

```text
templates\
  Договоры\
    Уступка.docx
  Заявления\
    Заявление в суд.docx
```

Внутри Word-файла добавляйте переменные в фигурных скобках:

```text
Должник: {Должник}
ИНН: {ИНН должника}
Заказчик: {Заказчик}
```

## Работа со словарём переменных

Словарь переменных помогает FillDoc понимать, что разные написания в шаблонах могут означать одно и то же поле.

Например, в разных Word-шаблонах может встречаться:

```text
{ИНН должника}
{ИНН ДОЛЖНИКА}
{ИНН Должник}
```

Для пользователя это одно поле. Словарь позволяет связать все эти варианты с одной переменной `ИНН должника`, чтобы FillDoc корректно находил данные в Excel, показывал недостающие поля и заполнял документы.

Пользовательский словарь хранится в файле:

```text
%USERPROFILE%\.filldoc\variables.json
```

Обычно редактировать этот файл вручную не нужно. Используйте вкладку `Переменные -> Словарь`.

### Что видно в разделе

В левой части раздела находится список переменных словаря. Для каждой переменной отображается:

- название поля;
- техническое имя;
- тип поля;
- признак обязательности.

В правой части находится карточка выбранной переменной:

- техническое имя;
- отображаемое название;
- группа;
- тип поля;
- обязательность;
- aliases;
- комментарий;
- список шаблонов, где используется эта переменная.

### Как добавить новую переменную

1. Откройте вкладку `Переменные`.
2. Перейдите во внутреннюю вкладку `Словарь`.
3. Нажмите кнопку `+`.
4. Заполните карточку переменной.
5. Нажмите кнопку сохранения.

Рекомендуемый пример заполнения:

```text
Техническое имя: inn_debtor
Название: ИНН должника
Группа: debtor
Тип поля: text
Обязательная: да
Комментарий: ИНН юрлица или ИП
```

Техническое имя лучше делать коротким и стабильным. Оно нужно программе как внутреннее имя переменной. Название можно писать привычно для пользователя, например `ИНН должника`.

### Как добавить alias

Alias - это дополнительный вариант написания той же переменной в шаблонах.

1. Выберите переменную в списке.
2. В поле `Новый alias` введите вариант написания.
3. Нажмите кнопку `+` рядом с полем.
4. Проверьте, что alias появился в списке `Aliases`.
5. Нажмите кнопку сохранения.

Пример aliases для переменной `ИНН должника`:

```text
ИНН ДОЛЖНИКА
ИНН Должник
инн должника
```

После сохранения FillDoc будет считать эти варианты одной переменной.

### Когда добавлять переменную, а когда alias

Добавляйте новую переменную, если это новое поле по смыслу:

```text
Дата договора цессии
Номер исполнительного листа
Адрес регистрации должника
```

Добавляйте alias, если поле уже есть, но в шаблоне оно написано иначе:

```text
Должник
ДОЛЖНИК
Наименование должника
```

### Тип поля

Тип поля помогает описать, какие данные ожидаются:

- `text` - обычный текст;
- `multiline` - длинный текст, адрес, резолютивная часть;
- `select` - значение из набора вариантов;
- `date` - дата;
- `number` - число или сумма;
- `boolean` - да/нет.

Сейчас тип сохраняется в словаре и используется как описание переменной. В дальнейшем он может использоваться для более удобных форм ввода и проверок.

### Обязательные переменные

Отметка `Обязательная` показывает, что поле важно для документов. Используйте её для данных, без которых документ нельзя считать готовым: номер дела, должник, ИНН, сумма, дата решения и другие ключевые реквизиты.

### Использование в шаблонах

В блоке `Использование в шаблонах` FillDoc показывает, какие шаблоны используют выбранную переменную или её aliases.

Это помогает понять:

- где встречается переменная;
- можно ли безопасно переименовать поле;
- какие шаблоны зависят от старого написания;
- почему переменная появляется в списке недостающих полей.

Чтобы список использования был актуальным, в настройках должна быть указана папка шаблонов.

### Как словарь влияет на заполнение документов

Если в Excel есть колонка:

```text
ИНН должника
```

а в шаблоне написано:

```text
{ИНН ДОЛЖНИКА}
```

то после добавления `ИНН ДОЛЖНИКА` в aliases FillDoc поймёт, что нужно взять значение из поля `ИНН должника`.

Словарь используется при:

- проверке качества шаблонов;
- поиске недостающих полей;
- заполнении одного шаблона;
- пакетной генерации документов.

## Генерация документов

1. Откройте вкладку заполнения.
2. Нажмите обновление данных, чтобы перечитать проекты и шаблоны.
3. Выберите проект.
4. Выберите один или несколько шаблонов.
5. Сформируйте список полей.
6. Если часть полей отсутствует, заполните их в таблице.
7. Нажмите генерацию `.docx`.

Готовые документы появятся в папке выгрузки из настроек.

## Работа с настройками

Настройки сохраняются автоматически в локальном файле:

```text
%USERPROFILE%\.filldoc\settings.json
```

Если приложение открывается с неправильными путями или не видит файлы, проверьте этот файл или заново сохраните пути во вкладке настроек.

## Резервные копии Excel

Перед операциями записи FillDoc создает бэкап Excel-файла в папке:

```text
_filldoc_backups
```

Папка создается рядом с Excel-файлом. Это защита от случайной потери данных при сохранении, архивировании или удалении проектов.

## Частые проблемы

### PowerShell не находит Python

Проверьте:

```powershell
python --version
```

Если команда не работает, установите Python 3.11+ и добавьте его в PATH.

### Excel не сохраняется

Закройте Excel-файл в Microsoft Excel и повторите действие. Открытый файл может блокировать запись.

### Шаблон не виден

Проверьте, что файл имеет расширение `.docx`, не начинается с `~$` и лежит внутри выбранной папки шаблонов.

### Переменная не заполнилась

Проверьте, что имя внутри `{...}` совпадает с колонкой Excel или поддерживается словарем переменных.
